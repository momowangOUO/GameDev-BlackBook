# 協作指南

感謝您有興趣為《遊戲研發與發行全書》做出貢獻！這是一個旨在將遊戲開發知識「去魅」並「科學化」的開源項目。

## 如何貢獻

### 1. 提交 Pull Request (PR)
我們非常歡迎以下類型的 PR：
* **邏輯校準：** 修正書中關於數值模型、心理學理論的引用錯誤或邏輯漏洞。
* **案例更新：** 替換過時的遊戲案例，補充最新的行業動態（如最新的二次元游戲運營事故或成功案例）。
* **錯別字與語法：** 修正基礎的文本錯誤。

**PR 規範：**
* 請確保您的修改基於最新版本的 `main` 分支。
* 請在 PR 描述中簡要說明修改的原因（例如：「修正了關於霍桑效應的誤用」）。

### 2. 提出 Issue
如果您對書中的某個核心觀點有異議，或者建議新增章節，請先在 Issues 區發起討論。

## 致謝
所有被採納的貢獻者，我們都將在未來的版本更新中，於首頁或致謝名單中保留您的 ID。讓我們一起建立更精準的行業知識庫。
