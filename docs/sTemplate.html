Level 03

核心循環與回饋

「遊戲是一連串有趣的選擇。」 —— 席德·梅爾 (Sid Meier)

1. 定義核心循環 (Core Loop)

在我們開始編寫任何程式碼之前，必須先問自己一個問題：玩家在遊戲中這 30 秒內主要在做什麼？這就是所謂的核心循環。

一個成功的動作遊戲核心循環，通常包含以下三個關鍵要素（項目符號範例）：

  • 行動 (Action)： 玩家進行操作，例如攻擊、跳躍或施法。
  • 回饋 (Reward)： 遊戲給予立即反應，例如傷害數字、擊殺音效或掉落物。
  • 擴展 (Expansion)： 玩家利用回饋來強化自己，準備迎接更強的挑戰。

2. 實作步驟詳解

要實現完美的打擊感，我們不能只依賴動畫。請依照以下順序檢查你的戰鬥系統（編號清單範例）：

  1. 檢查攻擊判定的時間點是否與動畫揮擊幀同步。
  2. 在命中瞬間添加 0.1 秒的頓幀 (Hit Stop)。
  3. 根據攻擊力度，施加對應強度的螢幕震動 (Screen Shake)。
  4. 播放含有高頻與低頻層次的打擊音效。

3. 程式碼實作：震動回饋

以下是一個簡單的 Unity C# 範例，展示如何在攻擊命中時觸發震動：

// 簡單的相機震動邏輯
public void TriggerShake(float duration, float magnitude) {
    StartCoroutine(Shake(duration, magnitude));
}
